Iris0fLyrics 开发手册（完整修订版）
1. 项目概述
Iris0fLyrics 是一款基于 Web 技术的一站式 TTML 歌词编辑器，支持逐字着色、色彩资产库、空间节点可视化编排、AI 辅助 After Effects 脚本生成等功能。项目采用 React + TypeScript + Vite 构建前端，使用 Jotai 管理状态，Radix UI 提供基础组件，Framer Motion 实现动画，Wavesurfer.js 渲染音频波形，并通过 Tauri 打包为桌面应用。

2. 技术栈
类别	技术
核心框架	React 19, TypeScript 5.9
构建工具	Vite 7
状态管理	Jotai 2.16 (with immer, history, optics)
UI 组件库	Radix UI, Fluent UI React Icons
动画	Framer Motion 12
音频处理	Web Audio API, Wavesurfer.js, 自定义 Web Worker

歌词核心	@applemusic-like-lyrics 系列包
国际化和多语言	i18next, i18next-parser
持久化	localStorage, IndexedDB (idb)
快捷键	自定义 hook + localStorage
分词	hyphen 库 + 自定义算法
桌面端	Tauri 2 (Rust 后端)
错误监控	Sentry
分析	Vercel Analytics, Speed Insights

4. 核心模块详解
4.1 歌词编辑模块 (lyric-editor)
功能：歌词行/单词的增删改查、时间轴显示、编辑模式下的各种操作。

关键文件：

components/lyric-line-view.tsx：LyricLineView 组件，渲染单行歌词，包含单词列表、翻译行、音译行，支持拖拽排序。

components/lyric-word-view.tsx：LyricWordView 组件，渲染单个单词，根据模式（简单/高级）显示不同的编辑控件。

components/roman-word-view.tsx：RomanWordView 组件，显示/编辑单词的罗马音。

components/lyric-line-menu.tsx：歌词行右键菜单。

components/lyric-word-menu.tsx：单词右键菜单。

components/sync-keybinding.tsx：SyncKeyBinding 组件，监听打轴快捷键并更新歌词时间戳。

utils/lyric-states.ts：提供获取当前选中位置、查找下一个单词等辅助函数。

utils/normalize-line-time.ts：根据单词时间戳重新计算行开始/结束时间。

tools/ReplaceWordDialog.tsx：替换单词对话框。

tools/TimeShift.tsx：批量平移时间对话框。

重要函数：

useCurrentLocation()：获取当前选中的行和单词信息。

findNextWord()：查找下一个可打轴的单词。

normalizeLineTime()：同步行时间与单词时间。

handleKeyDown 在 sync-keybinding.tsx 中处理 F/G/H 等打轴按键。

技术实现：

使用 jotai 管理歌词数据 (lyricLinesAtom) 和选择状态 (selectedLinesAtom, selectedWordsAtom)。

支持拖拽排序（基于原生 Drag & Drop API）。

单词编辑支持纯文本输入或高级模式（每个属性独立输入框）。

行号显示逻辑：背景行不递增行号，主行才递增。

技术实现
1. 空间节点与转场规则
空间节点采用相对坐标（百分比），导出时转换为 AE 滑块控制，确保跨分辨率适配。

支持“引流阀”和“分离轴”转场：通过 TransitionConfig 定义 type（follow/hold/delay）和 ratio，在导出时生成对应的关键帧插值逻辑，并在画布上通过 AETransitionCanvasPoint 组件可视化调整 ratio 触发点。

2. 多角色数据管理
多角色数据存储为字典 spatialDataMapAtom，键为角色 ID，值为该角色的空间数据（main/sub/ruby 轨道）。

角色资产库 roleSystemAtom 存储文件夹和角色名称，支持多文件夹管理，与节点坐标数据分离，便于复用和切换。

3. 数学函数轨迹
方程解析：通过正则替换将 ^ 转为 **，补全隐式乘号（如 5(x) → 5*(x)），并自动注入 Math 对象；使用 new Function 动态生成函数并缓存，避免重复编译。

坐标系转换：引入 globalToLocal / localToGlobal 函数，在节点拖拽时将全局百分比坐标转换为数学局部坐标进行吸附计算，再将吸附结果映射回全局坐标，同时考虑缩放、旋转和平移。

角度校正：通过局部两点计算实际屏幕像素差，得出视觉角度，再叠加用户设置的 axisOffset（由 mathSnapFlip 决定 ±90°）和 mathSnapInvert（±180°），确保节点长边/高边正确贴合曲线，避免因屏幕长宽比导致的视觉扭曲。

防卡顿优化：生成 SVG 路径时，若 y 值超出 [-150, 250] 范围则自动断笔，防止方程发散导致路径过长引发卡顿；使用 vectorEffect="non-scaling-stroke" 确保虚线粗细不随缩放变化。

4. 预览重写
calculateTrackState 根据轨道节点类型（in、preFocus、focus、postFocus、out）和动画时长，为每句歌词计算独立的“生命周期”时间轴。

歌词的入场、演唱、退场阶段严格按照 animDurationMs 和 visibleDurMs 进行插值，允许多句歌词同时渲染，且正在演唱的歌词强制高亮（透明度 100%）。

性能优化：通过 useMemo 缓存解析后的歌词文本，避免重复计算；仅对处于存活期的歌词进行渲染。

5. 节点对齐增强
修改 Moveable 组件参数，显式开启 snapDirections 和 elementSnapDirections 的 center 和 middle 属性，实现节点间水平/垂直中线吸附。

为固定节点（入场/焦点/离场）提供“+ 加入”按钮，避免手动拖拽的繁琐操作。

6. AI 插件隔离
AI 生成的代码通过 new Function 动态执行，且只能访问导出的 ai_custom_easing 和 ai_custom_effects 钩子，无法影响核心模板逻辑，保证安全性。

7. 界面优化
移除顶部“基础提示词”按钮，保持标题居中对齐。

顶部卡片增加说明文字，明确“纯净基础版”定位，并移除无用的“特效开关”。

在空间画布与 AI 工作台之间添加醒目的分割线和标题，区分“基础模式”与“进阶模式”。

数学轨迹控制区采用独立行布局（X轴、Y轴、大小、旋转各占一行），底部放置“恢复默认”按钮，提升操作直观性。

8. 数据持久化
所有配置（包括数学轨迹相关字段）均存储在 aeConfigAtom 中，并通过 atomWithStorage 自动持久化到 localStorage。

9. 模板选择拦截与空间节点映射
交互设计：用户在下拉框中选择模板时，若模板包含 spatialMap（预设空间节点数据），系统不会立即切换，而是弹出 AlertDialog 警告，让用户决定是否覆盖当前画板。

状态管理：使用 pendingTemplateId 暂存目标模板 ID，弹窗关闭后再执行实际切换；isAlertOpen 控制弹窗显示。

两种选项：

“仅切换模板”：仅将 selectedId 设为目标模板 ID，画板数据不变。

“清空并覆盖画板”：先通过 setSpatialDataMap 将模板的 spatialMap 写入空间数据原子，再切换模板 ID，并给出成功提示。

无数据模板：若模板不含 spatialMap，则直接切换，无任何打扰。

技术优势：此方案完全基于现有状态原子，无额外副作用，且利用 Radix UI 的 AlertDialog 组件实现美观安全的警告，杜绝了原生 confirm 在 Tauri 环境下可能失效的问题。
4.6 分词模块 (segmentation)
功能：将长单词自动拆分为多个音节，支持 CJK 字符拆分、西文按音节拆分、自定义规则。

关键文件：

utils/segmentation.ts：核心分词算法 segmentWord 和 segmentLyricLines。

utils/Transliteration/distributor.ts：罗马音自动分配算法（基于锚点对齐）。

utils/Transliteration/roman-warning.ts：检查罗马音一致性并标记警告。

utils/Transliteration/tokenizer.ts：罗马音分词器。

components/AdvancedSegmentation.tsx：高级分词对话框，允许用户配置规则和测试。

components/ManualWordSplitter.tsx：手动分词器组件。

states/index.ts：存储分词配置的原子（如 segmentationSplitCJKAtom）。

utils/hyphen-loader.ts：异步加载 hyphen 分词器。

重要函数：

segmentWord(word, config)：将单个单词拆分为多个单词，并重新分配时间。

predictLineRomanization(words, romanLyric)：将整行罗马音预测分配到每个单词。

loadHyphenator(lang)：异步加载指定语言的 hyphen 分词器。

applyRomanizationWarnings()：检查罗马音匹配并设置警告标记。

技术实现：

CJK 字符按字符拆分，西文利用 hyphen 库按音节拆分。

标点符号处理支持合并或独立。

自定义规则基于 Map 存储词到切分数组的映射。

7. 开发注意事项
颜色标记转换：导出 TTML 时，exportTTMLText 会智能将 #RRGGBB 和 {文本#RRGGBB} 转换为标准 tts:color 属性。

多角色兼容性：歌词行新增 role 字段，默认为 "1"，旧文件解析时也会兼容 isDuet 和 ttm:agent。

性能优化：频谱图使用虚拟滚动和瓦片缓存；歌词列表虚拟滚动；Jotai 原子细粒度订阅。

快捷键冲突：所有快捷键在输入框/可编辑元素内默认禁用，通过 isEditing 判断。

自动保存：基于项目 ID 聚合历史版本，保存间隔和保留数量可在设置中调整。

空间节点数据持久化：spatialDataMapAtom 与角色资产库分离，角色资产库存储在 roleSystemAtom 中，节点坐标存储在 spatialDataMapAtom 中（均使用 atomWithStorage）。

转场规则：TransitionConfig 中的 ratio 字段用于延迟转场的触发点，在画布上通过 AETransitionCanvasPoint 组件可视化调整。

AI 插件隔离：AI 生成的代码通过 new Function 动态执行，且只能访问导出的 ai_custom_easing 和 ai_custom_effects 钩子，无法影响核心模板逻辑。
