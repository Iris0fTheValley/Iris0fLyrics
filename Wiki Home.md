📖 Iris0fLyrics 使用指南
欢迎使用 Iris0fLyrics。本工具在继承了原版强大的 TTML 歌词打轴功能之上，重构了逐字上色系统与空间节点可视化引擎。旨在帮助你无需学习复杂的 After Effects（AE）操作，即可一站式生成极具设计感、高自定义的特效字幕视频。

声明：本指南的基础加载与打轴部分基于原项目 amll-ttml-tool 扩展编写。

# 1. 基础准备：导入、编辑与打轴
##  1.1 加载与编辑歌词
导入：通过左上角菜单 文件 -> 打开歌词 加载文件。系统不仅支持 TTML 格式，也支持直接导入纯文本、LRC、YRC、Lyricify Syllable 等格式。

拆分与排版：点击顶部的 编辑模式 按钮。你可以增加新行、对长句进行拆分或合并。注意：最终渲染会严格保留文本中的空格，请务必留意中英文之间的空格间隙。

多语言轨道：如果需要制作中日双语或带罗马音的字幕，请通过菜单 查看 -> 显示翻译/音译歌词 开启对应轨道。

# 1.2 音乐打轴 (核心快捷键)
点击上方的 打轴模式 按钮并加载音频文件即可开始同步时间戳。为了提升打轴效率，请熟练使用以下桌面端默认快捷键：

## 移动打轴光标
| 按键 | 说明 |
| :--- | :--- |
| A | 移动到上一个单词 |
| D | 移动到下一个单词 |
| W | 移动到上一行歌词的第一个单词 |
| S | 移动到下一行歌词的第一个单词 |

## 跳转播放时间
| 按键 | 说明 |
| :--- | :--- |
| R | 移动到上一个单词，并把播放进度设置为该单词的起始时间 |

## 记录时间戳（主要打轴按键）
| 按键 | 说明 |
| :--- | :--- |

| H | 将当前进度记录为当前单词的结束时间，然后移动到下一个单词（常用于结束当前句子、空出间奏区域，或是表现歌手停顿式的演唱） |

# 2. 核心进阶：逐字着色系统（Colorizer）
Iris0fLyrics 引入了非侵入式的颜色标记语法。为了兼容底层数据结构，主歌词与翻译/音译歌词的上色逻辑完全不同，请务必区分使用。

## 2.1 主歌词上色规则（原生分块法）
主歌词拥有独立的时间轴分词块（Word Blocks），不需要且不能使用花括号。

操作方法：选中你想要上色的分词块，在文本的末尾直接加上 #颜色代码 即可。

正确示例：鸢#FF0000 （AE 渲染时会自动隐藏颜色代码，仅展示红色的“鸢”字）。

##  2.2 翻译与音译歌词上色规则（花括号包裹法）
翻译和音译轨道属于整行文本，没有底层细分词块，因此必须使用花括号来限定上色边界。

操作方法：使用 {文本#颜色代码} 的语法包裹需要变色的部分。

正确示例：{鸢尾#FF0000}花 （渲染后“鸢尾”显示为红色，“花”为默认白色）。

## 2.3 悬浮调色工具栏的使用
为提升效率，我们提供了全局悬浮调色盘，建议在打轴完成后使用它进行高效上色：

智能替换机制：你可以直接用鼠标在原生取色器上滑动，选中的文字会实时刷新颜色代码。如果对已上色的文字（如 {字#FF0000}）再次使用调色盘，系统会智能替换原颜色代码，而不会产生 { {字#FF0000} #00FF00} 这种导致乱码的嵌套层。

资产库：右击调色盘上的圆形预设槽，可将当前颜色永久保存。点击“⚙️”可展开色彩资产管理库进行分组管理。

一键预设上色：你可以在文本框内使用光标选中文本，然后点击悬浮调色盘中保存的颜色块按钮，就能一键实现 鸢尾花 -> {鸢尾花#77BBDD} 的替换，自动将文本套入花括号并附加颜色代码。

## 2.4 标准 TTML 格式导出兼容 (重要说明)
无论是主歌词的 #颜色代码 还是翻译轨道的 {文本#颜色代码}，这仅仅是本软件编辑界面为了方便可视化操作而设计的“快捷语法”。
当你点击菜单 文件 -> 保存/导出 为标准的 .ttml 格式时，系统会在底层进行静默编译，将这些标记完美转化为 TTML 标准的属性标签（类似于 <span tts:color="#FF0000">）。它绝对不会把标记作为脏文本暴露出去，完美兼容所有支持标准 TTML 解析的第三方播放器。

# 3. 视觉编排：空间节点漫游引擎 (Spatial Engine)
通过左侧面板切换至“🌌 纯净版：基础空间动线引擎”，传统的数值输入排版将升级为“物理级空间拖拽画布”。此功能也旨在为 AE 高阶用户提供极其干净的物理工程底座。

## 3.1 相对坐标与节点概念
画布采用 0%-100% 的相对坐标系。你在网页中排版后，无论未来在 AE 中将合成修改为 1080p、4K 还是 9:16 竖屏格式，歌词的空间动线都会自动等比缩放，绝对不会跑偏出画。

入场点 / 离场点：控制单行歌词在屏幕中出现与消失的位置。

焦点 (Focus)：歌词正处于被演唱状态时停留的核心位置。

先焦点 / 次焦点 (Pre/Post-Focus)：未演唱或已演唱完毕时的停靠驿站。可无限叠加以构建极其复杂的漫游轨迹（如 Z 字形走位）。

## 3.2 边缘场景：古文竖排与磁吸对齐
多维排版架构：在左侧“参数实验室”中，可将“文本排版架构”切换为竖向排布（古文格式）。画布内的节点会物理变形成瘦高条。引擎底层注入了动态包围盒（sourceRectAtTime）计算，无论文字有多长，它的物理几何中心将永远死死咬住你设置的节点坐标。

十字磁吸：当节点靠近画布正中心或中轴线时，会自动触发物理磁吸对齐。节点与节点之间也会触发吸附，辅助强迫症用户完成完美居中。

## 3.3 异常排查：无法拖拽子轨道？（联动锁机制）
如果你发现翻译或音译的节点在画布中“死死卡住”无法拖拽，请检查右侧控制栏右上角的 “⚙️ 联动锁” 或是界面左上角的联动锁死开关：

检查是否开启了 🔒 锁定子轨道拖拽。该功能是为了防止在拖拽主轨道时误触子轨道。将其关闭即可自由拖拽。

在大部分情况下，建议保持 跟随主轨道移动 处于开启状态。这样你只需要排版主歌词，翻译就会自动保持相对距离跟随漫游。

## 3.4 参数实验室与预设管理
在 AE 导出界面的左侧设有“动效与特效参数实验室”，你可以在此硬编码全局的合成大小、字体、字间距、文字描边等基础视觉参数。如果你调配出了一套非常满意的参数环境，可以通过顶部的 “📤 导出预设” / “📥 导入预设” 按钮将其保存为本地 JSON 文件，方便在不同的工程中一键恢复你的专属视觉配置。

# 4. 魔法注入：AI 辅助与插件化组装
系统采用了“物理骨架 + AI 特效插件”的控制反转工作流。彻底杜绝了让 AI 盲目生成大量代码导致排版崩溃或代码截断的痛点。

## 4.1 核心操作流程
参数硬编码：在左侧“参数实验室”调整好字体名称（必须与 AE 内字体名完全一致）、字号、合成大小。

生成提示词：在底部的“ 第一步：给大模型的提示词”面板中，分别用自然语言描述你想要的动效（如：“像弹簧一样飞入”）与视觉特效（如：“距离中心越远高斯模糊越强”）。点击复制按钮。

对话与组装：将复制的内容粘贴给主流大模型，如 ChatGPT / Claude / Gemini / DeepSeek。将 AI 返回的两个简短的 JavaScript 插件函数粘贴回面板右侧的“⚡ 第二步：粘贴 AI 生成的代码”输入框。

一键导出：点击下方的组装导出按钮，即可获得一份能够直接在 AE 中运行的 .jsx 完美脚本。

原理解析：AI 只被允许通过底层暴露的 API 接口修改运动速度曲线（贝塞尔缓动）和挂载滤镜，它绝对没有权限，也无法修改你在画布上精心画出的运动轨迹坐标。

💡 防污染隔离机制：顶部的常规导出按钮独立于底部的 AI 工作台，两者处于严格的物理与逻辑隔离状态。如果你在上方导出纯净版脚本，它绝对不会被你在下方输入框中粘贴的任何 AI 插件代码所影响或污染。

## 4.2 ⚠️ 已知局限性与未来规划（关于模板分享）
目前的 AI 插件化工作流采用的是“前端直接拼装并导出最终 JSX 脚本”的模式。这意味着，你调教出的一套极品特效代码，目前只能在网页底部的输入框内单次生效并导出，暂时失去了如同旧版本那样打包成 .js 模板文件并分享给社区其他用户的能力。

我深知“模板生态与分享”对于创作者社区的重要性，这一妥协是暂时的。在未来的版本更新中，将重构模板模块，支持将“AI 特效代码”固化保存、导入，并恢复作为独立模板在社区间流转的功能。

# 5. 常见问题与注意事项 (FAQ)
## 5.1 After Effects (AE) 兼容性问题
版本要求：由于导出的脚本底层使用了原生的 sourceRectAtTime 动态计算以及三阶贝塞尔数组引擎，强烈建议使用 After Effects CC 2019 或更高版本。过低版本的 AE 可能会由于旧版 JavaScript 引擎不支持而抛出数组维度错误。（注：AE 版本繁多，可能存在其他未知的 API 兼容性问题）。

多语言报错免疫：本引擎已针对中文、日文等多语言版 AE 进行了底层绝对索引重构，不会再出现旧版插件常见的找不到“Point/点”的系统报错。

## 5.2 画面撕裂、排版偏移或乱码
请确保在“参数实验室”中设置的“AE 字体名”与你电脑中安装的字体名称完全一致（精确到大小写和空格，例如必须写 Microsoft YaHei 而不是 微软雅黑）。否则 AE 可能会回退到系统默认字体，导致字间距、宽度和几何中心计算出现严重偏差。

请确保歌词文本不要包含无法解析的不可见控制字符。
